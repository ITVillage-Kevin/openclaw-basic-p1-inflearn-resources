# OpenClaw.ai 기초 1편 수강생 리소스
OpenClaw.ai 기초 1편 강의에서 사용하는 소스코드, 실습 파일, 템플릿, 슬라이드, 가이드 문서를 모아둔 저장소입니다.

이 저장소는 강의 수강 중 필요한 자료를 **수업 순서대로 쉽게 찾을 수 있도록** 구성되어 있습니다.  
특히 Docker 설정, `.env` 설정, 테스트 문서, 결과 예시 파일 등을 단계별로 제공하여 강의를 따라가며 직접 실습할 수 있게 했습니다.

## 저장소 구조

```bash
.
├─ docs/         # 무료 가이드 문서 등 보충 자료
├─ lesson-files/  # 수업별 실습용 파일
├─ releases/      # 수업 단계별 완성 상태의 실행 파일
├─ samples/      # 결과 예시 및 샘플 데이터
├─ slides/       # 강의에서 사용한 슬라이드 자료
└─ templates/    # 재사용 가능한 템플릿 파일
```
⭐ 각 폴더에는 각 폴더의 용도와 활용 방법을 설명한 README.md 파일이 각각 포함되어 있으니, 활용 전 꼭 읽어주시길 바랄게요!

## 폴더별 안내

### docs/
무료 가이드 문서를 모아둔 폴더입니다.
설치 체크리스트, Docker 명령어 요약, 트러블 슈팅 가이드 같은 보조 자료를 여기서 확인할 수 있습니다.

<br/>


### lesson-files/
- 각 수업에서 사용하는 실습용 파일을 모아둔 폴더입니다.

- 설정 파일, 테스트용 문서, 분석 대상 파일, 샘플 입력값처럼 수업 진행에 필요한 자료를 수업 단위로 제공합니다.

<br/>


### releases/
- 실습을 진행하면서 자동으로 생성되는 설정 파일이나 결과물 등이 포함된 폴더입니다.

- 예를 들어 OpenClaw 초기 부팅을 하고 나면 자동으로 설정들이 추가되는 openclaw.json 파일 등을 의미합니다.

<br/>


### samples/
- 실습 결과를 이해하기 위한 예시 출력물과 샘플 데이터를 모아둔 폴더입니다.

<br/>


### slides/
- 강의에서 사용한 슬라이드 자료를 모아둔 폴더입니다.

- 인프런 강의 자료로 올라가는 내용과 함께 참고할 수 있도록 정리했습니다.

<br/>


### templates/
openclaw.json, 시스템 프롬프트, 사용자 프로필 템플릿처럼 반복해서 활용하는 기본 템플릿을 모아둔 폴더입니다.

<br/>


## 사용 방법
- 수업별로 강의 영상을 시청합니다.
- docs/에서 수업별로 필요한 문서의 내용들을 확인합니다(선택).
  - 01-section01-class04-OpenClaw 설치 전 필수 체크리스트 (사양 및 환경).pdf
  - 02-section02-class06-Docker 설치 가이드.pdf
  - 03-section02-class09-Docker 기본 명령어 모음.pdf
  - 07-Ubuntu 기본 명령어 및 환경변수 설정 가이드
    - 이 문서에서 **환경 변수 설정** 부분은 Section 6의 보안 특강을 마친 후에 참고하시기 바랍니다.
- 해당 수업의 lesson-files/를 열어 실습에 필요한 파일을 확인합니다.
  - lesson-files/ 하위에는 수업별로 실습에 필요한 자료들이 포함되어 있습니다.
- releases/에서 현재 수업 단계에 맞는 설정이나 결과물을 확인합니다.
  - releases/ 하위에는 수업별로 실습을 완료한 시점의 최종 설정 및 결과물이 포함되어 있습니다.
- 필요하면 templates/와 samples/를 참고해 설정값과 결과 예시를 확인합니다.
- 강의 슬라이드는 slides/에서 확인합니다.

<br/>


## 권장 흐름
- 강의 영상 시청
- docs/ 확인
- lesson-files/ 준비
- releases/ 확인 및 적용
- samples/로 필요한 샘플 확인
- templates/로 설정 보완

<br/>


## 주의사항
- .env 파일은 실제 비밀값이 포함되지 않는 형태로 제공됩니다.
- 각 수업 자료는 강의 진도에 맞는 버전을 사용해야 합니다.
- releases/ 폴더의 파일은 수업 단계별 상태를 반영하므로, 중간 수업에서는 final 버전을 바로 사용하지 않는 것이 좋습니다.
