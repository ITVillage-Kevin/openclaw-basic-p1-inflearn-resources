# Releases 안내

이 폴더에는 **수업 진행 단계별로 완성된 설정 파일 스냅샷**이 들어 있습니다.  
실습이 꼬였을 때 특정 시점의 상태로 돌아가거나,  
“영상에서 설명한 결과 상태”를 바로 보고 싶을 때 참고용으로 사용합니다.

---

## Releases 폴더의 역할

- `lesson-files/`  
  - 각 수업을 **시작하기 직전 상태**의 파일 세트를 제공합니다.
- `releases/`  
  - 해당 수업(또는 섹션)의 실습을 끝냈을 때 기준이 되는 **완성본/중간 결과물**을 제공합니다.

즉,

> 실습을 시작할 때는 `lesson-files/`를 보고,  
> 실습이 끝난 뒤 결과를 확인하거나 꼬였을 때 복구할 때는 `releases/`를 봅니다.

---

## 폴더 구조 예시

```bash
releases/
├─ section03-lesson11/
│  └─ openclaw.json
├─ section03-lesson13/
│  └─ openclaw.json
├─ section06-lesson19/
│  ├─ .env
│  └─ docker-compose.yml
└─ final/
   ├─ .env
   ├─ docker-compose.yml
   ├─ openclaw.json
   └─ 기타 최종 상태 및 결과에 필요한 파일들
```

- sectionXX-lessonYY/

  - 해당 수업까지 진행했을 때의 “정상 동작하는 상태”를 담고 있습니다.

- final/

  - 기초 1편 전체를 마쳤을 때 기준이 되는 최종 완성본입니다.

실제 폴더 이름과 구성은 강의에서 안내하는 규칙에 따라 변경될 수 있습니다.

---

## 언제 releases를 사용하면 좋을까요?
- 실습 중 설정이 꼬였을 때

  - “어디가 잘못됐는지 모르겠다” 싶으면, 현재 섹션/수업 번호에 해당하는 releases/sectionXX-lessonYY/의 파일을 참고해 비교하거나 그대로 덮어쓸 수 있습니다.

- 코드를 직접 다 타이핑하지 않고 결과 상태를 먼저 보고 싶을 때

  - 시간이 부족하거나, 먼저 전체 흐름을 보고 싶은 경우 `releases`의 설정 파일로 컨테이너를 띄운 뒤, 나중에 여유 있을 때 `lesson-files`를 따라가며 복습할 수 있습니다.

- 이전·다음 단계와의 차이를 확인하고 싶을 때

  - 예: `openclaw.min.json` → `openclaw.json` 온보딩 이후 상태 비교

  - 예: 섹션 6 보안 특강에서, 환경변수 적용 전/후 `docker-compose.yml` 비교

---

## 사용 방법 요약
1. 인프런에서 현재 또는 목표로 하는 섹션/수업 번호를 확인합니다.

2. `releases/sectionXX-lessonYY/` 폴더를 엽니다.

3. 내부 `README.md`를 먼저 읽어, 이 스냅샷이 어떤 상태를 의미하는지 확인합니다.

4. 필요하다면:
  - 해당 폴더의 설정 파일을 복사해 **내 작업 디렉터리에 덮어쓰기** 하거나

  - 내 파일과 나란히 열어 **어디가 다른지 비교**해 봅니다.

---

## 주의사항
- `releases` 폴더에 있는 파일들은 **예시이자 기준점**입니다. 실제 여러분 환경의 경로, 포트, 키 값 등은 상황에 따라 달라질 수 있습니다.

- 민감한 정보(API 키, 비밀번호 등)는 환경변수로 관리하는 것이 원칙이므로, `releases`에도 평문 키 값은 포함하지 않습니다.

- “중간 상태”가 의미 없는 수업(설명만 있는 수업, 설정 변화가 없는 수업)은 굳이 `releases` 폴더를 만들지 않을 수 있습니다.

> “잘 돌아가는 상태를 캡처해 둔 세이브 포인트 모음”

이라고 생각하시면 됩니다.
막혔을 때 부담 없이 참고하고, 비교하고, 필요하면 그대로 가져다 쓰셔도 좋습니다.
