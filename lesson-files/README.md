# Lesson Files 안내

이 폴더에는 **각 수업을 시작하기 직전 상태**에서 필요한 실습 파일들이 들어 있습니다.  
즉, 인프런 강의에서 해당 레슨을 시작할 때 기준이 되는 초기 파일 세트를 모아둔 공간입니다.

---

## 폴더 구조

예시는 다음과 같은 형태를 가정합니다.

```bash
lesson-files/
├─ section01-lesson01/
├─ section01-lesson02/
├─ section02-lesson05/
├─ section02-lesson07/
├─ section02-lesson08/
├─ section03-lesson10/
└─ ...
```

## sectionXX-lessonYY 구성

섹션 번호(sectionXX)와 레슨 번호(lessonYY)는 인프런 강의 커리큘럼과 1:1로 대응합니다.

각 폴더 안에는 해당 레슨을 시작하기 직전에 준비되어 있어야 하는 파일들만 포함됩니다.

### 폴더 예시

```bash
section02-lesson08/
├─ README.md
├─ config/
│  └─ (레슨 시작 전 기준이 되는 설정 파일들)
├─ workspace/
│  └─ (실습에 사용할 입력 데이터, 문서 등)
└─ 기타 레슨에서 언급하는 초기 파일들
```

### 내부 구성

- **config/**: OpenClaw나 Docker 설정 관련 초기 상태 파일
- **workspace/**: 해당 레슨에서 다룰 테스트 문서, 입력 데이터, 예시 파일 등
- 기타 레슨에서 “미리 준비되어 있다”고 가정하는 추가 파일

## Lesson Files와 Releases의 차이

- **lesson-files/**: 실습을 시작하기 직전 기준 상태
  - “지금 레슨을 시작하려면 여기 있는 파일들만 있으면 된다”는 개념
- **releases/**: 실습을 끝냈을 때 또는 강의에서 설명한 상태까지 진행된 완성/중간 결과물
  - “영상에서 설명한 결과 상태를 바로 보고 싶다” 혹은 “중간에 꼬여서 특정 단계 상태로 점프하고 싶다”는 경우 사용

> 헷갈리면 이렇게 생각하세요
>
> - `lesson-files/` = 내가 직접 실습을 시작하기 위한 준비물
> - `releases/` = 강의를 다 따라 했을 때의 완성본/중간 저장본

## 사용 방법

1. 인프런에서 현재 보고 있는 섹션/수업 번호를 확인합니다.
2. 같은 번호의 `sectionXX-lessonYY/` 폴더를 연 뒤, 내부 파일들을 내 작업 폴더로 복사합니다.
3. 강의에서 안내하는 대로 파일을 수정하거나 명령어를 실행하면서 실습을 진행합니다.
4. 실습 결과가 잘 되었는지 확인하고 싶다면, `releases/`의 동일 레슨 또는 다음 단계 폴더와 비교해 봅니다.
