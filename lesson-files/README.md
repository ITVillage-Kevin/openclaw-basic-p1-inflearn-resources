# Lesson Files 안내

이 폴더에는 **각 수업을 시작하기 직전 상태**에서 필요한 실습 파일들이 들어 있습니다.  
즉, 인프런 강의에서 해당 레슨을 시작할 때 기준이 되는 초기 파일 세트를 모아둔 공간입니다.

---

## 폴더 구조

예시는 다음과 같은 형태를 가정합니다.

```bash
lesson-files/
├─ section01-lesson01/
├─ section01-lesson02/
├─ section02-lesson05/
├─ section02-lesson07/
├─ section02-lesson08/
├─ section03-lesson10/
└─ ...
```

## sectionXX-lessonYY 구성

섹션 번호(sectionXX)와 레슨 번호(lessonYY)는 인프런 강의 커리큘럼과 1:1로 대응합니다.

각 폴더 안에는 해당 레슨을 시작하기 직전에 준비되어 있어야 하는 파일들만 포함됩니다.

### 폴더 예시

```bash
section02-lesson08/
├─ config/
│  └─ (수업 시작 전 기준이 되는 설정 파일들)
├─ workspace/
│  └─ (실습에 사용할 입력 데이터, 문서 등)
└─ 기타 수업에서 언급하는 초기 파일들
```

### 내부 구성

- **config/**: OpenClaw 설정 관련 초기 상태 파일
- **workspace/**: 해당 수업에서 다룰 테스트 문서, 입력 데이터, 예시 파일 등
- 기타 수업에서 “미리 준비되어 있다”고 가정하는 추가 파일

## Lesson Files와 Results의 차이

- **lesson-files/**: 실습에 필요한 파일
  - “실습을 정상적으로 완료하려면 여기 있는 파일들만 있으면 된다”는 개념
- **results/**: 실습을 끝냈을 때 진행된 완성 결과물
  - “영상에서 설명한 결과 상태를 바로 보고 싶다” 는 경우 사용

> 헷갈리면 이렇게 생각하세요
>
> - `lesson-files/` = 이 파일들만 있으면 실습이 정상적으로 진행된다는 준비물
> - `results/` = 강의를 다 따라 했을 때 자동으로 생성되는 결과물

## 사용 방법

1. 인프런에서 현재 보고 있는 섹션/수업 번호를 확인합니다.
2. 같은 번호의 `sectionXX-lessonYY/` 폴더를 연 뒤, 내부 파일들을 내 작업 폴더로 복사합니다.
3. 강의에서 안내하는 대로 파일을 수정하거나 명령어를 실행하면서 실습을 진행합니다.
4. 실습 결과가 잘 되었는지 확인하고 싶다면, `results/`의 동일 레슨 또는 다음 단계 폴더와 비교해 봅니다.
