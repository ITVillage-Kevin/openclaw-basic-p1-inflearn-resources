# Section 2 Lesson 7 – 초기 폴더 구조 준비

이 폴더는 **섹션 2, 수업 7: "OpenClaw 프로젝트 폴더 생성과 역할 이해"**를 시작하기 직전 상태를 담고 있습니다.

이 단계의 목표는 **폴더 구조와 파일 자리만 먼저 만들어 두는 것**이며,  
내용은 이후 레슨에서 하나씩 채워 나가게 됩니다.

---

## 포함된 내용

```bash
section02-lesson07/
├─ config/
├─ workspace/
├─ .env
└─ docker-compose.yml
```

- config/
  - 아직 아무 파일도 없습니다.
  - 앞으로 OpenClaw 설정 파일 등이 들어갈 설정용 폴더입니다.

- workspace/
  - 아직 아무 파일도 없습니다.
  - 나중에 분석할 문서, 테스트 데이터 등 실제 작업 파일이 들어가는 폴더입니다.

- .env
  - 내용이 비어 있는 상태로 제공됩니다.
  - 추후 레슨에서 환경 변수(예: 포트, API 키 등)를 채워 넣게 됩니다.

- docker-compose.yml
  - 내용이 비어 있는 상태로 제공됩니다.
  - 이후 레슨에서 서비스 설정(Docker 이미지, 볼륨, 포트 등)을 한 줄씩 채워 넣게 됩니다.

---

## 어떻게 사용하면 되나요?
이 폴더 안의 파일/폴더를 **강의에서 안내하는 위치(예: demo/작업 디렉터리)**로 그대로 복사합니다.

현재 수업(섹션 2, 수업 7)에서는 **“이 구조가 왜 이렇게 필요한지”**와 각 폴더의 역할을 이해하는 데 집중합니다.

실제 내용 채우기(.env 내용, docker-compose.yml 설정 등)는 다음 수업에서 단계적으로 진행합니다.

이 단계에서 중요한 것은 파일 내용이 아니라 구조 자체입니다.
지금은 “빈 껍데기”만 준비해 두고, 뒤 수업에서 하나씩 살을 붙인다고 생각하면 됩니다.
