# Section 4 수업 13 – 경로 최적화와 자아 설정

이 폴더는 **섹션 4, 수업 13: "넌 이제 내 비서야 – 경로 최적화와 자아 설정"**을 진행할 때 참고용으로 사용하는 파일과 변경 사항을 설명합니다.

이 수업의 핵심 목표는 두 가지입니다.

1. `openclaw.json`에서 **workspace 경로를 실제 사용 환경에 맞게 수정**한다.  
   - 기존: `/home/node/.openclaw/workspace`  
   - 수정: `/home/node/workspace`

2. OpenClaw 대시보드 채팅창에서 대화를 통해  
   **`IDENTITY.md`, `USER.md` 내용이 자동으로 업데이트되는 경험**을 해 본다.

---

## 이 레슨에서 다루는 파일들

이 수업에서는 새로운 설정 파일을 “처음부터 새로 만드는 것”보다,  
**이전 수업에서 만들어진 파일을 수정·업데이트**하는 작업이 중심입니다.

### 1. `openclaw.json`

- **이전 수업(섹션 3 수업 11) 완료 시점**의 `openclaw.json`을 기반으로 합니다.
- 수강생이 직접 수정해야 하는 핵심 포인트는 다음 한 줄입니다.

```jsonc
// 기존 값 (예시)
"workspace": "/home/node/.openclaw/workspace"

// 수정 후 값
"workspace": "/home/node/workspace"
```

- 이 변경을 통해:

  - Docker 컨테이너 내부의 작업 경로가

  - 호스트와 볼륨 마운트한 실제 workspace 폴더와 더 일관되게 맞춰집니다.

⚠️ 이 수업에서는 openclaw.json 전체를 교체하지 않고, 이전 단계에서 생성된 파일을 열어 일부 필드만 수정하는 것이 중요합니다.

### 2. `IDENTITY.md`, `USER.md`
- 이 두 파일은 보통 아래와 같은 경로(예시)에 위치하게 됩니다.

  - workspace/IDENTITY.md

  - workspace/USER.md

- 섹션 4 수업 13에서는:

  - OpenClaw 대시보드의 채팅창에 메시지를 입력하면서

  - “내 비서의 성격, 역할, 나에 대한 정보”를 대화 형식으로 전달합니다.

  - 그 결과가 IDENTITY.md, USER.md에 자동으로 반영됩니다.

이 파일들은 **직접 수동으로 편집하는 예시**가 아니라,
“대화를 통해 AI가 스스로 자기소개서와 사용자 프로필을 작성하는 형태”를 경험하는 데 초점이 있습니다.

---

## 이 수업에서 제공되는 것 / 제공되지 않는 것
- 제공되는 것:

  - 어떤 필드를 어떻게 수정해야 하는지에 대한 **설명과 예시** (강의 및 문서에서 안내)

- 제공되지 않는 것:

  - 온보딩 이후에 자동으로 채워진 완성된 IDENTITY.md, USER.md 샘플 파일은 개인 정보·취향 등이 강하게 반영되는 파일이므로, 별도의 정답 파일을 강제하지 않습니다.

필요하다면, 강의 보조용으로:

- samples/identity-example/IDENTITY.md

- samples/identity-example/USER.md

와 같은 예시용 템플릿/샘플을 따로 제공할 수 있습니다.
하지만 이 수업의 핵심은 “각 수강생이 자기만의 내용을 채운다”는 점입니다.

---

## 정리: 이 수업 이후의 상태
섹션 4 수업 13을 마치면, 이상적인 상태는 다음과 같습니다.

1. `openclaw.json` 안의 workspace 경로가 `/home/node/workspace`로 올바르게 수정되어 있다.

2. OpenClaw 대시보드 채팅을 통해 대화한 내용이 `IDENTITY.md`, `USER.md`에 잘 반영되어 있다.

3. 이후 수업에서 폴더 탐색, 파일 요약, 보고서 생성 등을 할 때 “나를 잘 아는 나만의 비서”처럼 동작할 준비가 된 상태가 된다.
