# Section 3 Lesson 11 – Hello OpenClaw! 첫 번째 부팅과 시스템 점검

이 폴더는 **섹션 3, 수업 11: "Hello OpenClaw! 첫 번째 부팅과 시스템 점검"**을  
진행하기 위해 필요한 초기 설정 파일과, 수업이 끝났을 때 참고할 수 있는 최종 설정 파일을 안내합니다.

이 레슨의 핵심 목표는:

- 최소 설정만 담긴 **`openclaw.min.json`** 을 기반으로
- 온보딩(대화 기반 초기 설정)을 진행하고
- 그 결과가 반영된 **최종 `openclaw.json`** 상태를 이해하는 것입니다.

---

## 초기 상태: openclaw.min.json

수업을 시작할 때 필요한 최소 파일은 아래와 같습니다.

```bash
section03-lesson11/
└─ openclaw.min.json
openclaw.min.json
```

- OpenClaw를 첫 부팅할 수 있을 정도로만 채워진 최소 설정 파일입니다.
  - `openclaw.min.json` 파일의 파일명을 `openclaw.json`과 같이 변경해서 사용하세요.

  - 필수 필드들만 포함되어 있으며, 사용자 정보나 세부 동작 규칙 등은 비워져 있거나 기본값 상태입니다.

  - 이 레슨에서는 이 파일을 기반으로 컨테이너를 띄우고, 온보딩 과정을 통해 실제 openclaw.json 내용을 채워 나가게 됩니다.

👉 수업을 시작할 때는 openclaw.min.json만 있으면 됩니다.(파일은 강의에서 안내한 config/ 폴더 위치에 파일명을 변경한 후, 복사해 두세요.)

---

## 레슨 진행 중 하는 일
이 수업에서 수강생은 대략 다음과 같은 흐름으로 실습을 진행합니다.

1. Docker 컨테이너를 부팅하고 OpenClaw를 실행한다.

2. 터미널 또는 UI에서 OpenClaw와 대화하며 온보딩 질문에 답한다.

3. OpenClaw가 사용자의 답변을 바탕으로 내부 설정 파일(openclaw.json)을 갱신한다.

4. 첫 부팅 로그를 확인하고, 시스템이 정상적으로 동작하는지 점검한다.

이 과정에서 초기 `openclaw.json`은 “빈 그릇” 역할, 실습 후 생성·업데이트되는 `openclaw.json`은 “내 환경에 맞게 채워진 실제 설정” 역할을 하게 됩니다.

---

## 최종 상태: openclaw.json (Release 안내)
수업 11이 끝난 뒤의 최종 결과물은 **온보딩을 마친 뒤 자동으로 채워진 `openclaw.json`**입니다.

- 이 완성된 예시 파일은 releases/section03-lesson11/ 폴더에서 확인할 수 있습니다.

최종 openclaw.json 예시는:

- 최소 설정(openclaw.min.json)에서 어떤 필드가 추가·수정되었는지,

- 온보딩 질문에 어떤 정보가 반영되는지

를 이해하는 데 도움을 주기 위한 참고용 샘플입니다.
실제 수강생의 환경에서는 각자의 답변에 따라 내용이 달라질 수 있습니다.

---

## 어떻게 사용하면 되나요?
1. 이 폴더의 `openclaw.min.json`을 config/ 폴더 아래에 복사한 후, 파일명을 `openclaw.json`으로 변경합니다.

2. 강의 안내에 따라 컨테이너를 실행하고, **OpenClaw**와의 대화를 통해 온보딩을 마칩니다.

3. 온보딩 후 생성·업데이트된 openclaw.json 파일을 열어, 최소 설정과 어떤 차이가 있는지 비교해 봅니다.

4. 비교가 어려울 경우, releases에 제공된 예시 openclaw.json과 나의 파일을 나란히 열어 차이를 확인해 보세요.
