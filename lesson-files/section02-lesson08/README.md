# Section 2 Lesson 8 – 볼륨 마운트까지 설정된 초기 상태

이 폴더는 **섹션 2, 수업 8: "볼륨 마운트: 내 PC와 AI 작업실 안전하게 연결하기"**를  
시작할 때 기준이 되는 파일들을 담고 있습니다.

이 단계에서는 **Docker 컨테이너와 로컬 폴더(config, workspace)를 볼륨 마운트로 연결**하는 것이 목표입니다.

---

## 포함된 내용

```bash
section02-lesson08/
├─ .env
└─ docker-compose.yml
```

- .env

  - docker-compose.yml에서 참조하는 환경 변수들이 정의되어 있습니다.

  - 예: 컨테이너 이름, 포트 번호, 볼륨 경로 등 (실제 키 이름과 값은 강의 내용을 따릅니다).

- docker-compose.yml

  - OpenClaw 관련 컨테이너를 띄우기 위한 기본 서비스 정의가 들어 있습니다.

  - 로컬 PC의 config/, workspace/ 폴더를 컨테이너 내부 경로와 연결하는 볼륨 마운트 설정까지 포함되어 있습니다.

  - 이 파일은 “섹션 2 수업 8 기준 완성본”이며, 이후 수업에서 추가 설정이 더해집니다.

---

## 사전 조건
이 레슨을 시작하기 전에 아래가 준비되어 있어야 합니다.

1. 섹션 2, 수업 7에서 안내한 대로:

  - 작업 디렉터리에 config/ 폴더가 존재

  - 작업 디렉터리에 workspace/ 폴더가 존재

  - 빈 .env, docker-compose.yml가 있던 상태에서 이 폴더의 파일로 교체 (또는 덮어쓰기)

2. Docker Desktop이 설치되어 있고 실행 중이어야 합니다.

---

## 어떻게 사용하면 되나요?
1. 이 폴더(section02-lesson08/) 안의 .env, docker-compose.yml 파일을 강의에서 안내한 작업 디렉터리로 복사합니다.
2. 필요하다면 .env 안의 값(예: 포트 번호, 경로 등)을 강의에 맞게 수정합니다.
3. 터미널에서 해당 작업 디렉터리로 이동한 뒤, 강의에서 안내하는 명령어를 실행합니다.
  - 예: docker compose up -d
4. 컨테이너가 정상 실행되는지, config/, workspace/ 폴더가 올바르게 마운트되었는지 확인합니다.
